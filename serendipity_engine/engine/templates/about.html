About

Why interop?

Interoperability is an amplifier of value.

By enabling more applications to use existing digital resources, the community gains more value from those resources.

For example, a poem in an online Emily Dickinson collection can be used to make a point in a collection of 19th century scientific papers, and then annotations within both collections can help make sense of an application exploring the intersection of literature and science.

Interoperability enables a library's works to flourish.

What is interop?

Interoperability is a property of a system. That system is embedded in (and helps enable) a community that supports it.

In an interoperable library system library content, data, metadata, and services can be easily used and re-used by multiple software applications. Each use can then be re-used by other applications, creating a virtuous circle.

This is achieved by community support of open standards in a distributed system.

Interoperability is always a matter of degree. It is never perfect and has no end point. Indeed, the Harvard Library system is already highly interoperable.

Paths forward

The Library Interoperability Initiative plans on moving down several paths simultaneously, always with the engagement and support of the Harvard Library community, and beyond:

  • Self-knowledge: Improve the community's awareness of its efforts, so we can better learn from one another.
  • Lower the hurdles: Make it easier for more of what the Library knows to be used, reused, found, and enhanced. Do this quickly
  • Community research: Support the community's efforts to develop more comprehensive projects and standards over the longer term

Action items for the first year:

  • Community: Increase self-knowledge, and create a working group
  • Cloud server: Implement an initial platform that will provide some interoperability services, tools, and a place for experimentation.
  • Guidelines: Voluntary guidelines for the development of services, standards, and APIs.
